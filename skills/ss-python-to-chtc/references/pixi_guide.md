# Pixi for Python Projects

## What is Pixi?

Pixi is a modern package manager for Python (and other languages) that provides:
- **Reproducible environments**: Lock files ensure exact same dependencies everywhere
- **Fast installations**: Uses pre-compiled conda-forge binaries
- **Cross-platform**: Single tool works on Linux, macOS, Windows
- **Multi-language**: Supports Python, R, C++, and more
- **Task runner**: Built-in task management

## Why Pixi Over pip/venv?

### Traditional pip workflow:
```bash
python -m venv venv
source venv/bin/activate  # Different on Windows!
pip install -r requirements.txt
python app.py
```

### Pixi workflow:
```bash
pixi run start
```

### Key Advantages

**Reproducibility**
- pip: `requirements.txt` often lacks system dependencies
- Pixi: `pixi.lock` captures everything including C libraries, Python version, OS packages

**Speed**
- pip: Compiles from source for many packages (slow, requires build tools)
- Pixi: Uses pre-built conda-forge binaries (fast, no compilation)

**Environment Management**
- pip: Manual venv creation and activation
- Pixi: Automatic environment management, no activation needed

**System Dependencies**
- pip: Must manually install system libs (apt-get, brew, etc.)
- Pixi: Handles system dependencies automatically

## Pixi.toml Structure

```toml
[project]
name = "my-app"
version = "0.1.0"
description = "My application"
channels = ["conda-forge"]          # Where to get packages
platforms = ["linux-64", "osx-64"]  # Target platforms

[dependencies]
python = "3.11.*"
pandas = ">=2.0"
numpy = "*"
fastapi = ">=0.100"

[pypi-dependencies]
# Use PyPI for packages not in conda-forge
some-pypi-only-package = "*"

[tasks]
start = "python app.py"
test = "pytest"
format = "black ."
lint = { cmd = "ruff check .", depends_on = ["format"] }

[feature.dev.dependencies]
pytest = "*"
black = "*"
ruff = "*"
```

## Common Commands

### Installation and Setup
```bash
# Install Pixi
curl -fsSL https://pixi.sh/install.sh | bash

# Initialize new project
pixi init

# Install dependencies
pixi install
```

### Managing Dependencies
```bash
# Add package from conda-forge
pixi add numpy

# Add with version constraint
pixi add "pandas>=2.0"

# Add from PyPI
pixi add --pypi requests

# Add to dev feature
pixi add --feature dev pytest

# Remove package
pixi remove numpy

# Update all packages
pixi update

# Update specific package
pixi update numpy
```

### Running Tasks
```bash
# Run defined task
pixi run start

# Run multiple tasks
pixi run lint test

# Run task from feature
pixi run --feature dev test
```

### Environment Management
```bash
# Activate shell (like venv)
pixi shell

# Run command in environment
pixi run python script.py

# List environments
pixi list
```

## Docker Integration

### Multi-stage Build Pattern
```dockerfile
# Build stage
FROM ghcr.io/prefix-dev/pixi:latest AS builder
WORKDIR /app
COPY pixi.toml pixi.lock ./
RUN pixi install --locked

# Runtime stage
FROM ubuntu:22.04
WORKDIR /app
COPY --from=builder /app/.pixi /app/.pixi
COPY app.py .
ENV PATH="/app/.pixi/envs/default/bin:$PATH"
CMD ["python", "app.py"]
```

### Benefits
- **Small images**: Only runtime dependencies in final stage
- **Fast builds**: Cached layers for dependencies
- **Reproducible**: Lock file ensures consistency

## Features System

Features allow optional dependency groups:

```toml
[feature.dev.dependencies]
pytest = "*"
black = "*"

[feature.docs.dependencies]
sphinx = "*"
sphinx-rtd-theme = "*"

[feature.ml.dependencies]
tensorflow = "*"
scikit-learn = "*"
```

```bash
# Install with feature
pixi install --feature dev

# Run with feature
pixi run --feature ml train

# Activate shell with feature
pixi shell --feature dev
```

Pixi ensures dependencies work on all specified platforms.

## Lock Files

`pixi.lock` contains:
- Exact versions of all packages
- Package hashes for verification
- Dependency tree
- Platform-specific resolutions

**Never edit manually**. Generated by `pixi install` or `pixi add`.

## Channel Priority

```toml
channels = ["conda-forge", "bioconda"]
```

Searches channels in order. `conda-forge` has 20,000+ packages and is well-maintained.

## PyPI Fallback

When a package isn't in conda-forge:

```toml
[pypi-dependencies]
my-custom-package = ">=1.0"
```

Or via CLI:
```bash
pixi add --pypi my-custom-package
```

## Best Practices

### Always commit pixi.lock
Lock file ensures reproducible builds. Commit it to version control.

### Use version constraints
```toml
python = "3.11.*"      # Any 3.11.x
pandas = ">=2.0,<3"    # SemVer range
numpy = "*"            # Latest (use sparingly)
```

### Separate dev dependencies
```toml
[feature.dev.dependencies]
pytest = "*"
```

### Define tasks
```toml
[tasks]
test = "pytest tests/"
format = "black ."
```

### Pin Python version
```toml
python = "3.11.*"  # Not "3.*"
```

## Troubleshooting

### Package not found
- Check if it's in conda-forge: https://anaconda.org/conda-forge
- Use PyPI fallback: `pixi add --pypi package`

### Slow installs
- Use `pixi install --locked` to skip solving
- Check network connection to conda-forge

### Lock file conflicts
```bash
# Regenerate lock file
rm pixi.lock
pixi install
```

### Platform issues
- Ensure target platform in `platforms = [...]`
- Check package supports your platform on anaconda.org

## Migration from pip

### From requirements.txt
```bash
# For each package in requirements.txt
pixi add package-name

# Or convert programmatically
cat requirements.txt | grep -v '^#' | xargs -I {} pixi add {}
```

### From environment.yml (conda)
```bash
pixi init --import environment.yml
```
